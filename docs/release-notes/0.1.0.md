# PLC Collector 0.1.0 Release Notes

Release date: 2026-07-22

## Summary

PLC Collector 0.1.0 is the first MVP release. It provides a modular runtime for
validated PLC/OPC UA tag configuration, deterministic local mock reads,
asynchronous OPC UA batch reads, per-connection worker isolation, bounded
poll-result queuing, relational persistence, durable offline spool support,
read-only health/runtime APIs, service-process helpers, and bundled license
credits.

## Compatibility

- Python: 3.12 or newer.
- Local development storage: SQLite.
- Real PLC connectivity: OPC UA through `asyncua`.
- Service deployment: external process manager such as systemd, WinSW, or NSSM.

## Known Limitations

- The long-running service host prepares process paths, logging, PID files, and
  graceful stop handling, but full production orchestration still depends on an
  external service manager.
- Configuration is loaded from JSON and environment overrides; encrypted secret
  storage is out of scope for this MVP.
- The API is read-only and exposes runtime/health information only.
- SQLite is the validated local persistence backend for MVP testing.
- OPC UA integration tests use a local asyncua server fixture; production PLC
  interoperability must be validated against target devices before deployment.
