# Task 16: Release MVP

## Cel

Przygotować pierwsze spójne wydanie MVP.

## Zakres

- pełny test suite,
- test długotrwały,
- test awarii połączenia,
- test awarii storage,
- test graceful shutdown,
- aktualizacja README,
- finalizacja changelogu,
- version bump,
- release notes,
- tag Git,
- artefakt dystrybucyjny,
- lista znanych ograniczeń.

## Kryteria akceptacji

- wszystkie testy przechodzą,
- lint i type check przechodzą,
- dokumentacja instalacji jest kompletna,
- licencje są kompletne,
- changelog zawiera wszystkie zmiany,
- wersja jest spójna w kodzie i paczce,
- release nie zawiera sekretów ani danych testowych klientów.

## Proponowany commit

```text
chore(release): prepare version 0.1.0
```
